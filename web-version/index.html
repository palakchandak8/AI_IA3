Z-score normalisation before clustering
K-Means works by measuring distances between points. If income ranges 15–137 and score ranges 1–99, income values are larger just because of scale — not meaning — and would dominate the clustering. The code subtracts the mean and divides by the standard deviation for both columns, rescaling each to mean ≈ 0, std ≈ 1.
function normalize(arr) { const mean = arr.reduce((a,b) => a+b, 0) / arr.length; const std = Math.sqrt(arr.reduce((a,b) => a+(b-mean)**2, 0) / arr.length) || 1; return arr.map(v => (v - mean) / std); } const incS = normalize(RAW.map(r => r[2])); // income column → z-scores const scS = normalize(RAW.map(r => r[3])); // score column → z-scores const pts = incS.map((v, i) => [v, scS[i]]); // 2D points for clustering // Example: income 16, mean=60.6, std=25.6 → z = (16-60.6)/25.6 = -1.74

K-Means clustering (K=5, pure JavaScript)
The algorithm runs 15 times with random starting centroids and keeps the run with the lowest inertia (tightest clusters). After convergence, cluster IDs are remapped to stable semantic names based on which quadrant each centroid falls in — so the same cluster always means the same thing across page reloads.
const {labels: raw, cents} = kmeans(pts, 5); // 15 runs, best wins // Remap random cluster IDs → stable quadrant labels const quad = c => { const [x, y] = c; if (x < -0.3 && y > 0.3) return 0; // Low income, High spend if (x > 0.3 && y > 0.3) return 4; // High income, High spend if (x < -0.3 && y <= -0.3) return 2; // Low income, Low spend if (x > 0.3 && y <= -0.3) return 3; // High income, Low spend return 1; // Middle cluster }; const labels = raw.map(l => remap[l]); // labels[i] = 0..4 for each of the 200 customers

Age, Income & Score histograms — manual binning
There's no built-in histogram type in Chart.js, so the code manually builds bin counts. It defines bin edges as an array, then loops every row and places the column value into the correct bin using a nested for loop. The same pattern is used for all three histograms — only the column index and bin edges differ.
// Age histogram (r[1]) const bins = [18, 25, 30, 35, 40, 45, 50, 55, 60, 65, 70]; const counts = new Array(bins.length - 1).fill(0); RAW.forEach(r => { for (let i = 0; i < bins.length - 1; i++) { if (r[1] >= bins[i] && r[1] < bins[i+1]) { counts[i]++; break; // stop once placed } } }); // Income uses r[2], Spending Score uses r[3] — same pattern
The bin label strings (e.g. "18-25") are generated by bins.slice(0,-1).map((v,i) => v+'-'+bins[i+1]).
Segment cards & profile chart — per-cluster averages
For each of the 5 clusters, the code filters RAW to just the rows assigned to that cluster, then computes averages for income, score, age and female %. These feed both the segment cards above and the "Avg Income & Score per Cluster" bar chart.
for (let c = 0; c < 5; c++) { // Keep only rows K-Means assigned to this cluster const m = RAW.filter((_, i) => labels[i] === c); const avgIncome = (m.reduce((s,r) => s + r[2], 0) / m.length).toFixed(0); const avgScore = (m.reduce((s,r) => s + r[3], 0) / m.length).toFixed(0); const avgAge = (m.reduce((s,r) => s + r[1], 0) / m.length).toFixed(0); const femalePct = Math.round( m.filter(r => r[0] === 1).length / m.length * 100 ); // → renders segment card + feeds grouped bar chart }
